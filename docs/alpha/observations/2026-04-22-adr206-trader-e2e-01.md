# Observation 01 — verify.py invariants are pre-ADR-205/206

**Date**: 2026-04-22
**E2E run**: post-purge ADR-206 trader validation (run 1 of 1)
**Persona**: alpha-trader
**Phase**: pre-run harness check
**Severity**: medium (friction, not blocker — harness is advisory, not gating)

## What happened

Running `python3 scripts/alpha_ops/verify.py --all` against the post-purge,
post-ADR-206 workspace returned 4/23 passing checks for alpha-trader and
3/20 for alpha-commerce.

The FAILs are structurally correct consequences of the architecture changes,
not broken state:

- `agent_count: got 1, expected 12` — ADR-205 collapsed the signup roster to 1 (YARNNN).
- `essential task 'back-office-*' missing` — ADR-206 Phase 1 deletes signup-scaffolded
  back-office tasks; they auto-materialize on trigger.
- `essential task 'daily-update' missing` — ADR-206 makes daily-update opt-in.
- `core file /workspace/IDENTITY.md missing` — ADR-206 relocated to
  `/workspace/context/_shared/IDENTITY.md`.
- Same for `/workspace/BRAND.md`, `/workspace/AWARENESS.md`,
  `/workspace/CONVENTIONS.md` — all relocated.
- `context domain 'trading' files missing` — ADR-205 directory registry
  collapse + ADR-206: domain directories materialize on first agent write.

## Classification

**A (system/architecture).** The alpha harness is a versioned artifact
(per execution-discipline reminder #10). Its `expected` invariants in
`personas.yaml` predate ADR-205 and ADR-206.

## Proposed fix

Update `docs/alpha/personas.yaml` `expected` blocks to reflect the ADR-206
post-purge signup invariants:

```yaml
alpha-trader:
  expected:
    agent_count: 1                         # was 12 (ADR-205)
    active_bots: [thinking_partner]        # was [trading_bot, thinking_partner] (bots are lazy, ADR-205)
    essential_tasks: []                    # was 5-element list (ADR-206)
    scaffolded_tasks: []                   # was ["trading-sync"] (ADR-206)
    platform_connections: 1                # unchanged (preserved through purge)
    core_files:
      - /workspace/context/_shared/IDENTITY.md
      - /workspace/context/_shared/BRAND.md
      - /workspace/context/_shared/CONVENTIONS.md
      - /workspace/memory/awareness.md
      - /workspace/review/IDENTITY.md
      - /workspace/review/principles.md
    # risk_md_path + context domain checks deferred until operation is declared
```

Then `verify.py` regains informational value: 23 → ~10-12 checks, all
relevant to the post-ADR-206 clean-slate.

## What this teaches about the architecture

The harness and the substrate must stay coupled. The ADRs were the hardest
part of the refactor — but the harness is the feedback lever that tells us
whether the substrate matches our intent. We should not ship an ADR that
mutates signup invariants without also updating `personas.yaml`.

Future discipline: **every ADR that mutates `workspace_init` invariants
must also mutate `personas.yaml` expected blocks in the same commit**.
Add to CLAUDE.md execution reminders.

## Did it block the E2E?

No. verify.py is advisory, not gating. E2E proceeds.
