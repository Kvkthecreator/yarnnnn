## Session discipline notes

Observations worth preserving for next-session calibration:

1. **Canon-first works but must close with validation.** Today's session produced the taxonomy sharpening cleanly. But the gap between canon maturity and ground-truth validation is real. Don't start a new canon-first session without a validation phase committed upstream.

2. **Same-day renames require clear sequencing.** The `agent_framework.py` → `agent_registry.py` → `agent_orchestration.py` → `orchestration.py` chain worked but was audit-correction heavy. When doing a rename pass under a canon flip, write the canon flip first, then do the rename once from the target.

3. **Enum-slug exceptions need explicit documentation.** The `"specialist"` and `"platform-bot"` enum strings cross Python + TS + revision data; ripping them out would be a multi-day coordinated change. GLOSSARY exceptions table is the right place. Future canon flips that touch enum values should decide upfront whether they're renaming the enum or the concept.

4. **Historical ADRs are frozen.** Do not rewrite ADR content under later canon. New ADRs (212 supersedes portions of 194/211) are how supersession is recorded. Same for prompts CHANGELOG entries.

5. **Auto-mode needs firm scoping.** Multiple times today the session risked scope-creep (extra renames, extra files touched) and was held in check by the mid-session "are these files agent orchestration?" interjection. For long sessions, periodic regrouping (like this handoff) is the discipline.
