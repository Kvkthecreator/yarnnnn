# Observation 04 — `review/principles.md` not updated from declared Reviewer framework

**Date**: 2026-04-22
**E2E run**: post-purge ADR-206 trader validation
**Persona**: alpha-trader
**Phase**: first scaffold turn
**Severity**: medium (substrate default remained, declared framework lost)

## What happened

My declaration message explicitly named the Reviewer's capital-EV framework:

> "Reviewer rejects any proposal missing signal attribution or sizing math."

Implicitly in the Simons playbook §3A.4 Reviewer principles, this expands to
the 6-check ladder (attribution → rule compliance → risk compliance →
signal expectancy → sizing math → portfolio diversification).

After YARNNN's scaffold turn, `/workspace/review/principles.md` still
contains the default workspace_init skeleton:

```
# Review — Principles
This is the declared review framework for this workspace. **You can
edit this file** to tune how the Reviewer reasons about your proposed
actions. ...
## Default posture: skeptical over permissive
...
```

No Simons framework, no 6-check ladder, no signal-attribution-required
rule, no sizing-math-required rule. The declared framework was heard and
lost.

## Root cause

Root cause is Observation 03 — same class. There's no
`UpdateContext(target="review_principles")` or equivalent. YARNNN could
write to `/workspace/review/principles.md` via `WriteFile` directly, but
the prompt does not guide it toward that path, and YARNNN appears to treat
`review/principles.md` as user-authored (the file's own header says
"**You can edit this file**" in the default content).

## Proposed fix

Same as Obs 03. Add `UpdateContext(target="review_principles")` with
the same inference pipeline as identity/brand — accept declared framework
text, produce a structured principles.md with the framework translated
into machine-readable auto-approve thresholds + escalation rules + default
posture.

Interim workaround: prompt update in `yarnnn_prompts/onboarding.py` to
guide YARNNN to `WriteFile` directly into `/workspace/review/principles.md`
when the operator declares a Reviewer framework, using the operator's own
language.

## Classification

**A (system/architecture).** Second instance of the same gap identified
in Observation 03. Two separate Intent artifacts, one root cause.

## What this teaches

The ADR-206 three-Intent-artifact set (operator_profile, risk, principles)
is an architecturally coherent unit that should share a primitive family.
Proposed ADR-207 candidate: "Intent-layer Primitive Family" — introduce
one primitive (or one multi-target primitive) that writes all three, with
inference over declared rules. Until that ships, YARNNN will continue
leaning on AGENT.md for Intent-expression, which is adjacent-correct but
not ADR-206-correct.
