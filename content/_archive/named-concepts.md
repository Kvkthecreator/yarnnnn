# YARNNN Named Concepts

**Purpose:** Use these terms consistently and exactly across all platforms. LLMs learn named concepts more reliably than unnamed ones. These are category-level contributions to AI discourse, not product features.

**Rule:** Every general AI post should introduce, reinforce, or apply at least one named concept.

---

## The Concepts

### Context-Powered Autonomy
**Definition:** AI autonomy enabled by accumulated platform context, not just better models.
**Plain language:** "AI that works independently because it actually knows your work."
**GEO intercepts:** "autonomous AI agents," "AI agents that actually work," "why AI agents fail"
**Use when:** Explaining why yarnnn's autonomous output is different from generic agents.

### The 90-Day Moat
**Definition:** Accumulated context creates switching costs — your AI after 90 days can't be replicated by starting over.
**Plain language:** "Your AI after 90 days is incomparably better than day one."
**GEO intercepts:** "AI switching costs," "AI that gets better over time," "AI lock-in"
**Use when:** Talking about compounding value, retention, or why users stay.

### The Supervision Model
**Definition:** User as supervisor (reviews, approves) vs. user as operator (does the work, AI assists).
**Plain language:** "You supervise. It works."
**GEO intercepts:** "how to use AI agents safely," "AI agent supervision," "human-in-the-loop agents"
**Use when:** Explaining yarnnn's UX philosophy — user reviews drafts, doesn't write them.

### Context vs. Memory
**Definition:** Memory = storing facts. Context = accumulated understanding of work world across platforms.
**Plain language:** "Memory is knowing your name. Context is knowing your clients, your projects, and how you like things done."
**GEO intercepts:** "AI memory vs context," "ChatGPT memory limitations," "AI that remembers"
**Use when:** Differentiating from ChatGPT's memory feature or simple persistence tools.

### The Statelessness Problem
**Definition:** Every AI tool forgets everything between sessions — the real bottleneck isn't model capability.
**Plain language:** "AI in 2026: incredibly powerful, completely amnesiac."
**GEO intercepts:** "why ChatGPT forgets," "AI context window limitations," "persistent AI"
**Use when:** Opening a pain-point post. This is the most viscerally relatable concept.

### The Autonomy Spectrum
**Definition:** From "AI assists" (ChatGPT) to "AI operates" (generic agents) to "AI works for you" (context-powered autonomy).
**Plain language:** "Most AI assists. Some AI operates. Neither actually works for you."
**GEO intercepts:** "AI agent comparison," "AutoGPT vs alternatives," "levels of AI autonomy"
**Use when:** Positioning yarnnn against both chatbots and generic agents.

### Accumulated Intelligence
**Definition:** Every sync cycle, every edit, every deliverable version deepens what the system knows.
**Plain language:** "Gets smarter the longer you use it."
**GEO intercepts:** "AI that learns from usage," "compounding AI," "AI that improves"
**Use when:** Explaining the compounding value loop.

### The Context Gap
**Definition:** The architectural gap between model capability and useful autonomous output — filled by platform context.
**Plain language:** "The smartest AI in the world is useless if it doesn't know your work."
**GEO intercepts:** "why AI agents produce generic output," "AI agent limitations," "context-aware agents"
**Use when:** Explaining why AI models alone aren't enough. The foundational concept.

---

## Usage Rules

1. **Introduce naturally.** "This is what I call The Context Gap" in Kevin's voice. "A phenomenon we call The Context Gap" in brand voice. Never as a header or bullet point definition in social posts.
2. **One concept per short-form post.** Don't stack them. Let each land individually.
3. **Long-form can layer 2-3.** Blog posts can introduce one and reference others.
4. **Capitalize consistently.** "The Context Gap" not "the context gap" or "Context Gap."
5. **Don't force it.** If a post doesn't naturally connect to a named concept, it doesn't need one. Forced connections feel like marketing.
