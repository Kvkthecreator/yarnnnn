# Video Generation Skill — Implementation Plan

**Status:** Planning
**Date:** 2026-04-03
**Related:** ADR-118 (skills as capability layer), ADR-157 (visual assets)

## Context

The video render skill exists as scaffolding — Python script (`render.py`) calls Remotion CLI, but the **Remotion composition project doesn't exist**. Remotion CLI and Chromium are already in the Docker image. The skill is listed in agent capabilities (Marketing & Creative) but produces nothing.

## Design Principle: Agnostic Composition

The composition is **not** a set of hardcoded scene templates. It's a **slide-based renderer** — a sequence of slides, each containing positioned elements with entrance animations. The agent decides what content goes on each slide. The composition handles:

- **Timing** — slide sequencing, duration per slide, total video length
- **Transitions** — how slides enter/exit (fade, slide, cut)
- **Element animation** — how content elements appear within a slide (fade-in, spring, stagger)
- **Layout** — how elements are positioned (centered, split, grid)

This mirrors how presentation tools work: the creative decisions (what content, what structure) are upstream; the renderer handles mechanics.

## Composition Architecture

### Input: Slide-Based Props

```json

      "duration": 5,

      "elements": [
        {"type": "heading", "text": "Key Metric", "position": "left"},
        {"type": "value", "text": "$2.8M", "size": "3xl", "position": "right"},
        {"type": "badge", "text": "+33%", "color": "green", "position": "right"}
      ]
    },
    {
      "layout": "stack",
      "duration": 6,
      "transition": "fade",
      "elements": [
        {"type": "heading", "text": "Key Findings"},
        {"type": "list", "items": ["Cursor raised $400M Series C", "Zapier launched AI Actions", "Make acquired Integromat rebrand complete"]},
      ]
    },

        {"type": "text", "text": "Competitive Intelligence • Automated", "size": "sm", "color": "muted"}
      ]
    }

```

### Element Types (extensible)

Elements are the building blocks. They're simple, composable, and the set grows over time without changing the composition architecture:

| Element | Props | Rendering |
|---------|-------|-----------|
| `heading` | text, size (sm/md/lg/xl/2xl/3xl) | Text with font weight + size |
| `text` | text, size, color (foreground/muted/accent) | Body text |
| `value` | text, size | Large number display (e.g., "$2.8M") |
| `badge` | text, color (green/red/yellow/blue) | Pill-shaped indicator |
| `list` | items[] | Bulleted list with stagger animation |
| `divider` | — | Horizontal line |
| `spacer` | height | Vertical spacing |

Future elements (no architecture change needed):
- `image` — embed a URL (favicon, generated image from assets/)
- `bar-chart` — data-driven animated bars
- `progress` — progress bar with animation

### Layout Modes

Layouts control how elements are positioned on the slide:

| Layout | Description |
|--------|-------------|
| `center` | All elements vertically centered, stacked |
| `split` | Left/right split (elements specify `position: "left"/"right"`) |
| `stack` | Top-aligned, elements flow down |
| `grid` | N-column grid (elements fill cells in order) |

### Transitions

How one slide replaces the previous:

| Transition | Description |
|-----------|-------------|
| `fade` | Cross-dissolve (default) |
| `slide-left` | Previous exits left, new enters from right |
| `slide-up` | Previous exits up, new enters from bottom |
| `cut` | Instant switch (no animation) |

### Theme

Colors and typography for the entire video:

```json
{
  "background": "#0f172a",
  "foreground": "#ffffff",
  "accent": "#3b82f6",
  "muted": "#94a3b8",
  "font": "Inter"
}
```

The theme can be derived from BRAND.md at the agent level — the video skill itself just receives the theme object.

## Technical Implementation

### Composition Project Structure

```
render/skills/video/composition/
  package.json           ← remotion, react, @remotion/cli
  tsconfig.json
  remotion.config.ts
  src/
    Root.tsx             ← RemotionRoot → MainComposition
    MainComposition.tsx  ← Maps slides[] → <Series> of <Slide> components
    Slide.tsx            ← Layout router + element renderer
    elements/
      index.tsx          ← Element type → component map
      Heading.tsx        ← Heading element with size variants
      TextElement.tsx    ← Body text element
      Value.tsx          ← Large value display with optional count-up
      Badge.tsx          ← Colored pill badge
      List.tsx           ← Bulleted list with stagger
      Divider.tsx        ← Horizontal rule
    layouts/
      Center.tsx         ← Vertically centered stack
      Split.tsx          ← Left/right columns
      Stack.tsx          ← Top-aligned flow
    transitions/
      index.tsx          ← Transition type → animation config
    theme.tsx            ← Theme context provider
    types.ts             ← Zod schema for props validation
```

### Rendering Pipeline

1. Agent calls `RuntimeDispatch(type="video", input={slides: [...], theme: {...}}, output_format="mp4")`
2. Python `render_video()` writes props to temp JSON file
3. `npx remotion render` called with props file, composition ID, output path
4. Remotion renders frame-by-frame via headless Chromium
5. Output MP4 read and returned as bytes
6. Uploaded to Supabase Storage, workspace file written with `content_url`

Node.js version check needed — Remotion requires Node 18+. Current Dockerfile installs via `apt` which may be older. May need to use `nodesource` PPA.

### Constraints

- **Max 30 seconds** — hard limit (cost + render time)
- **Max 8 slides** — prevents runaway complexity
- **No audio** — silent videos (avoids complexity + licensing)
- **1080p** — landscape (1920x1080) or portrait (1080x1920)
- **Render time** — expect 30-90 seconds for a 15-second video at 30fps
- **Timeout** — 180s (already set in RuntimeDispatch for video type)

## Cost Model

- **Compute:** Render time on Render starter instance (~$7/mo). Video rendering is CPU-intensive but infrequent.
- **Storage:** MP4 files ~1-5MB each. Supabase free tier.
- **Frequency:** Realistically 1-5 videos/week per workspace. Not a cost concern.

## Risks

1. **Node version** — Remotion needs Node 18+. Docker `apt` install may give Node 12. Fix: use nodesource PPA.
2. **Docker image size** — React + Remotion deps could add 200-300MB. Acceptable for a Docker service.
3. **Render time variance** — Complex slides or many elements could push past timeout. Mitigate with element/slide caps.
4. **Chromium stability** — Headless Chromium rendering can be flaky under memory pressure. Render starter has 512MB RAM — may need to monitor.
