Architecture
Two-phase execution: cheap review, expensive generation.
Proactive Insights uses a Haiku review pass to decide whether themes warrant a full Sonnet generation. Cost-efficient by design — most cycles observe, few generate.
Phase 1 — Review
Haiku scans platforms
8+ targeted searches across Slack, Gmail, Notion, Calendar. Identifies emerging themes. Makes a decision: generate, observe, or sleep.
~$0.002/cycle · 4 API calls · 25 sec
→
Decision Gate
Generate or accumulate?
Generate: Themes at inflection → trigger Sonnet

Observe: Themes present, not ripe → log forward-looking notes

Sleep: Nothing notable → check again next cycle
Observations accumulate in deliverable_memory
→
Phase 2 — Generation
Sonnet researches + writes
Full generation with WebSearch. Internal signals + external context woven into intelligence brief. "What I'm Watching" shows progressive learning.
~$0.05/generation · 6 tool rounds · only when warranted
Why this can't be replicated
ChatGPT Deep Research: user picks topic → one-shot report → done
Perplexity: web-only search → no internal signals
YARNNN: autonomous topic selection from your platforms → progressive accumulation → generates only when themes converge
Validated behavior
Haiku identified 3 real themes from platform data
Correctly chose "observe" (themes present, not at inflection)
