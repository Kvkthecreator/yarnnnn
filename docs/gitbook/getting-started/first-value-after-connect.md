# First Value After You Connect

YARNNN is designed so the first useful result shows up quickly, without a long manual setup flow.

## What should happen

After you connect **Slack or Notion**, YARNNN moves through a short sequence:

1. Discovers the sources available on that platform
2. Auto-selects a small set of high-signal defaults
3. Starts the first sync
4. Grounds the shared workspace context
5. Gives TP and your agents enough material to produce useful work

In practice, the first visible value is usually one of these:

- a grounded answer from Thinking Partner
- the next daily update
- the first task output you ask YARNNN to set up

## What you should do

Treat the first useful result as a calibration pass.

- Read it end to end
- Keep it as-is if it is already useful
- Edit it if the emphasis, tone, or structure is off
- Tell TP what to change next time

You do not need to perfect the system before seeing value. The product is built around supervised improvement.

## If the first result is too broad or too noisy

That usually means coverage should be tightened, not that the model is wrong.

- Remove low-value Slack channels or Notion pages
- Ask for a narrower audience or structure
- Start with one platform before adding more

## What happens after the first run

Once the first task is live:

- future syncs keep the context current
- each run adds more execution history
- your edits and feedback become signal for better future runs
- the system can create additional tasks as value becomes clear

The goal is not just one draft. The goal is a loop that compounds.
