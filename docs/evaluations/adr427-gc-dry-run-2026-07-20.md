## Root-set rationale

Keep every blob referenced by ANY revision row (not just heads) — trace +
revert need the whole chain, and the `blob_sha` FK makes referenced blobs
undeletable anyway. `data-ref` pins and `derived_from` edges cite PATHS whose
revisions are chain rows, so the same root set covers them. A 24h age guard
covers the write_revision blob-before-revision crash window.
