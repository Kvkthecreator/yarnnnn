# Observation 05 — Task scaffold works; three real bugs in the same turn

**Date**: 2026-04-22
**E2E run**: post-purge ADR-206 trader validation
**Persona**: alpha-trader
**Phase**: second scaffold turn (operator asks YARNNN to complete tasks + principles + domain intent files)
**Severity**: mixed (1 major success, 3 real bugs)

## What YARNNN got right

Called `ManageTask(action="create")` four times, producing the expected
task scaffold with proper cron schedules, objectives, and full per-task
substrate (`TASK.md`, `DELIVERABLE.md`, `awareness.md`, `feedback.md`,
`memory/steering.md`):

| Task | Schedule | Status |
|---|---|---|
| track-universe | `0 7,12,17 * * 1-5` | active ✓ |
| signal-evaluation | `0 8 * * 1-5` | active ✓ |
| pre-market-brief | `15 8 * * 1-5` | active ✓ |
| weekly-performance-review | `0 18 * * 0` | active ✓ |

The ADR-149 + ADR-181 task substrate pattern (TASK + DELIVERABLE +
awareness + feedback + steering) materialized correctly for every task.
ManageTask primitive + task scaffolding pipeline works end-to-end on the
post-ADR-206 substrate.

## Bug 1 — Duplicate task created with `-2` suffix

YARNNN called `ManageTask(create)` for `track-universe` **twice in the
same turn**. The backend's slug de-dup kicked in and created
`track-universe-2`. So the workspace now has five tasks, not four:
the duplicate is functionally identical but clutters the task list
and doubles scheduled runs.

**Root cause hypothesis**: YARNNN's mid-turn memory of "did I already
create this task?" is thin. It likely re-read its plan after the
EditEntity call and re-emitted the first task creation.

**Fix**: `ManageTask._handle_create` could check for an existing task
with the same `title` + `agent_slug` before creating, and return the
existing row if present (idempotent create). Or YARNNN prompt guidance
to do a list/check before batch-creating.

**Classification**: A-system. Idempotence gap in ManageTask.

## Bug 2 — Mid-turn WebSearch call to `https://raw.githubusercontent.com/`

YARNNN made a `WebSearch` tool call to `https://raw.githubusercontent.com/`
(the root — no repo, no path) which fetched the GitHub homepage. The
response was 9147 chars of GitHub marketing copy. Hallucinated call —
nothing in my request referenced GitHub.

**Root cause hypothesis**: YARNNN was trying to find example code or
reference docs and reached for WebSearch as a last-resort primitive
when it couldn't resolve an internal lookup.

**Fix**: WebSearch usage guidance in `tools.py` should be tightened —
only when the user explicitly requests external info or when a clear
URL is cited. Internal reference-lookup should use SearchFiles or
QueryKnowledge, not WebSearch.

**Classification**: A-system + B-product. Prompt calibration issue.

## Bug 3 — `SearchEntities(query="trading operator agent", scope="agent")` returns 0

Right after creating the Trading Operator agent, YARNNN searched for it
via SearchEntities and got zero results — even though the agent exists
in the DB with title "Trading Operator" and slug `trading-operator`.
`ListEntities(pattern="agent:*")` returned it correctly.

**Root cause hypothesis**: SearchEntities uses the semantic search index
(ADR-151 / migration 145's `idx_agents_title_fts` or similar). The index
may not be updated on agent insert, requiring a reindex cron, or the
search uses embedding vectors that are only populated asynchronously.

**Fix**: Verify agent search index freshness. SearchEntities on freshly-
created agents should succeed. If index rebuild is async, add a fallback
to ListEntities on zero-result searches, or index synchronously on
`ManageAgent(create)`.

**Classification**: A-system. Search index freshness bug.

## What this run teaches about the E2E process

Three real bugs in one scaffold turn. If we had kept iterating frontend
polish instead of running the E2E, none of these would have surfaced
for weeks. The E2E exercise is exactly the right priority at this
architectural maturity level.

**Classification summary**: 3 A-system observations (1 idempotence,
1 prompt-calibration, 1 search-index) + 1 functional success.
